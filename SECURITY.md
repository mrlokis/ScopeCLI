# Security Policy

## Supported Versions

I currently provide security updates only for the **latest version** of ScopeCLI.  
If you are using an older release, please update to the most recent version to ensure you have the latest security fixes.

| Version            | Supported          |
| ------------------ | ------------------ |
| 0.00.1-dev (latest)| :white_check_mark: |
| < 0.00.1           | :x:                |

> 💡 As the project evolves, this table will be updated to reflect stable releases.  
> For now, always use the version installed by the [official installer](https://github.com/mrlokis/ScopeCLI#-installation-windows-powershell).

---

## Reporting a Vulnerability

I take security seriously. If you discover a vulnerability in ScopeCLI, please help me by reporting it privately.

### How to report

- **Open a [GitHub Security Advisory](https://github.com/mrlokis/ScopeCLI/security/advisories)** – this is the preferred method. It allows us to discuss the issue confidentially.

### What to expect

- You will receive an acknowledgment of your report within **8 hours**.
- I will investigate and provide an estimated timeline for a fix.
- Once the issue is resolved, i will credit you (if you wish) in the release notes.

Please do **not** disclose the vulnerability publicly until i have had a chance to address it.

Thank you for helping keep ScopeCLI and its users safe!
